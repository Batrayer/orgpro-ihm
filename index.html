Qu'est-ce que Org-Pro ?

Org-Pro est un outil qui permet de simplifier la gestion de projet. Il utilise des fichiers au format OrgMode ayant la caractéristique d'être très complet.

Pourquoi utiliser Org-Pro ?

Simple d'utilisation, il vous permet d'utiliser un logiciel de gestion de projet en local sur votre machine. Les fichiers générés conservent la structure d'OrgMode, et peuvent donc être utilisés dans d'autres logiciels utilisant le même format.

Comment puis-je l'utiliser ?

Org-Pro est utilisable sur n'importe quel ordinateur sous Unix ou Windows. Le seul pré-requis est d'installer JAVA 8.

Fonctionnalitées disponible :

Partie Tache :


- (Constructeur) Tache(String title) : Construit un objet tâche à partir du titre donné en paramètre.
- (Constructeur) Tache(String title, Tache tache) : Construit un objet tâche à partir du titre donné en paramètre et définie la tâche créé comme dépendante de la tâche donné en paramètre.
- boolean ajoutCollaborateur(String val) : Permet d'ajouter un collaborateur à un tâche. Le paramètre correspond au nom/prénom du collaborateur. Ne peut pas contenir de ":". Return false si contient ":" sinon true.
- boolean supprimerCollaborateur(String val) : Permet de supprimer un collaborateur d'une tâche. Le paramètre correspond au nom/prénom du collaborateur. Ne peut pas contenur de ":". Return false si contient ":" sinon true
- setDependance(Tache tache) : Permet de définir la tâche comme dépendante de la tâche donné en paramètre.
- removeDependance() : Permet de supprimer les dépendances de la tâche.
- changeTitle(String title) : Permet de modifier le titre de la tâche.
- boolean changeState(State state) : Permet de changer le state d'une tâche et d'écrire le changement dans le logbook si cela respecte les rêgles du kanban. TODO -> ONGOING -> DONE -> CANCELLED. Return true si le changement de state à fonctionné sinon false.
- boolean nextState() : Permet de passer au state suivant. TODO -> ONGOING -> DONE. Return false si la tâche était déja dans un state fini (DONE ou CANCELLED) sinon true.
- ajoutTag(String tag) : Permet d'ajouter un tag à la tâche.
- supprimerTag(String tag) : Permet de supprimer un tag à la tâche.
- boolean ajoutDeadline(String deadline) : Permet d'ajouter une deadline à une tâche sous la forme "yyyy-mm-dd". Return false si la date est incorrect sinon true.
- boolean ajoutScheduled(String scheduled) : Permet d'ajouter une date scheduled à une tâche quand on prévoit de commencer à faire la tâche sous la forme "yyyy-mm-dd". Return false si la date est incorrect sinon true.
- boolean ajoutClosed(String closed) : Permet d'ajouter une date closed à une tâche quand la tâche est fini sous la forme "yyyy-mm-dd". Return false si la date est incorrect sinon true.
- boolean ajoutProperty(String name, String value, boolean constructor) : Permet d'ajouter une property à un tâche à partir de la clé (name) et de sa valeur (value). On ne peut pas modifier les property "ID, DEPENDENCE, COLLABORATOR, CLOCK" sauf si le paramètre constructeur vaut true. Return true si l'ajout est passé.
- boolean supprimerProperty(String name, boolean interne) : Permet de supprimer une property à partir de la clé (name). On ne peut pas modifier les property "ID, DEPENDENCE, COLLABORATOR, CLOCK" sauf si le paramètre interne vaut true. Return true si la suppression est passé.
- boolean ecritureFichier(String path, boolean append) : Permet d'écrire les données de la tâche dans un fichier donné en paramètre (path). Si le paramètre append vaut true l'écriture se fera à la fin du fichier sinon elle écrasera le fichier. Return true si l'écriture est passé sinon false.
- static List<Tache> lectureFichier(String path) : Permet de lire un fichier à partir du path. Return la liste de tâche récupérer dans le fichier sinon null.
- static boolean supprimerTache(List<Tache> taches, int numTache) : Permet de supprimer une tâche d'une liste en paramètre (taches) à partir du numéro de la tâche dans la liste (numTache). Si d'autre tâche dépende de la tâche à supprimer alors elles seront supprimer aussi. Return true si la suppression est passé sinon false.
- static boolean deleteDependanceListe(Liste<Tache> list, int numTache) : Permet de rendre indépendante une tâche si elle possède une dépendance en donnant le numéro de la tâche (numTache) dans la liste (list). La méthode va réorganiser la liste en mettant la tache à la fin de la liste. La tâche choisi ne peut pas devenir indépendante s'il existe dans la liste une tâche qui dépend d'elle. Return true si la suppression de dépendance est passé sinon false.
- String toString() : Retourne les info de la tâche comme elles sont écrit dans le fichier.

Partie Scrum :


- static Liste<Tache> listerTacheState(Liste<Tache> liste, State state) : A partir de la liste en paramètre (liste), la méthode va retourner une liste de tâche qui sont dans l'état donné en paramètre (state).
- static Liste<Tache> listerTacheScheduled(Liste<Tache> liste) : A partir de la liste en paramètre (liste), la méthode va retourner une liste de tâche qui ont leurs date scheduled passé et qui sont toujours dans l'état TODO.
